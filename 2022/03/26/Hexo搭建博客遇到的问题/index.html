Hexo搭建博客遇到的问题

2022-03-26
|
|
hits
本文共527字。

早几年就想用hexo自己搭建博客了，中途遇到阻碍，计划就搁置了。最近查资料，发现某C字开头的网站十篇有九篇是同质内容，还很多广告。就又想起自己搭建博客的念头，这里记录下搭建时遇到的问题和解决方法。

问题一：初始化hexo时报错：ExperimentalWarning: The fs.promises API is experimental

这个问题是使用hexo init命令，项目自动引入了最新的模块，但本地的node的版本不是最新的

问题二：将node升级到最新版本

小吐槽：为什么升级npm只要一行命令，升级node却这么麻烦。网上查到的方法大多过时不适用。
下面讲解我踩到的几个坑：
（1）查到一个通过安装n模块的来管理node的版本，操作后却发现这种方法是争对linux的。无效。
（2）查到使用npm install npm@latest -g，报错You should probably upgrade to a newer version of node as we
can’t make any promises that npm will work with this version.。因为我之前先升级了npm，导致npm的版本高于node，命令执行不了。
（3）查到得先卸载node和删除各种路径，再在网上下载版本进行安装。路径太多不想删。
解决方法：
直接在 https://nodejs.org/zh-cn/download/
下载对应安装包，点击安装（需要在原来的node.exe路径下），安装完成后即是最新版本。

问题三：将ssh公钥添加到github账户中报错 Key is invalid. You must supply a key in OpenSSH public key format github

直接粘贴复制，ssh文件会破坏格式。需要点开git.bash，用clip < ~/.ssh/id_rsa.pub命令，就成功了。

问题四：hexo启用主题后不生效

直接从github上拉主题的分支，我默认放在了建博客的文件夹下。其实这是错误的，得放在themes的文件夹下才生效，不然会报错找不到Html。

问题五：github地址和本地渲染不一致

将config.yml里delopy的branch: master改成branch: main即可解决
